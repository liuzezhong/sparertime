管理全公司的设计需求

来自产品开发的需求

一家互联网公司，最重要的设计需求来自产品部门。对于许多公司来说，一个功能的设计涉及到 iOS 、Android 和 Web 等不同平台的设计。每一个平台都有自己平台的 UI 风格和设计规范。

如何平衡不同产品线的设计工作，以保证功能在不同平台上同步上线，考验设计师的时间管理能力。Teambition 可以在一个面板里实现对所有平台的设计任务管理，可供参考。

Teambition 的设计团队建立了一个「设计」的项目，不论是产品团队的设计需求还是非产品团队的设计需求，都会在这里呈现出来。

将所有的设计需求按照平台分布。在看板上，一眼就能看到不同平台上的设计任进展，一目了然。

非产品部门的需求

在完成产品设计和把握的同事，设计部门往往需要同时了解并完成公司其他部门的设计需求：市场部的海报，运营部的产品手册、企业识别系统设计等。 这样的需求有时候也是极为重要且紧急的任务。这些任务是市场或其他部门单独完成的，需要专业设计师的配合。

没有明确流程的设计团队，很容易被各种各样的设计任务打断了自己的步调，严重影响了设计师的安排和思路。

有了明确流程的设计团队，设计总监会第一时间汇总这些需求，妥善地分配给设计师完成。安排时间给设计师完成这个任务，一来二往，耗费不少时间；设计师为了了解具体的设计意图，又需要和需求提出人讨论。这么一来，任务还没有开始做，时间却已经过去良多。

设计总监建立了一个单独的任务分组，来收集所有非产品团队的设计工作。每一个列表是一个部门的设计需求。

设计总监可以将任务分配给其他设计师，设定截止时间，关联起相关资料。对于一些比较大的任务，页可以通过建立子任务分开。

在设计团队内部，沟通和协作是不可或缺的。比如，当交互设计师完成线框稿后，要交给 UI 设计师制作高保真原型；文案作者定稿后，会交给平面设计师来完成设计工作。当这些工作进展和多个产品线、多个客户端的任务在一起时，对设计总监的确是个巨大的挑战。

快速整理同类任务

我们会给每个任务打上不同的「标签」，设计总监可以很容易地了解到各项任务的进展。

在「标签」页面中，你可以方便地将某个类别的任务挑选出来，查看这些任务的详情。

如下图所示，你也可以在任务板上勾选任务，快速了解在这个任务板上的同一类任务的进展。

标签可以给任务、文件、日程、分享、账目做标记。也就是说，标签功能可以帮助我们整合同一个项目中类似的信息，帮助你做好信息整理。

常常听到身边的设计师在抱怨：80% 的时间都用在了混乱的沟通上。在过去，除了常用的邮件、IM 工具之外，设计师还需要使用网盘、Skype等多种工具辅助沟通。这就导致了修改设计稿成了一件极为痛苦的事情。

而团队协作工具 Teambition 可以帮助设计师统一工作协作方法，大幅度减少工作协作成本。

设计稿管理

设计素材和设计稿的共享和存储是非常重要的事情。Teambition 提供不限容量的文件库，允许设计师上传海量设计素材和设计稿。

文件讨论

设计师完成初稿后，将图片上传到文件库，并添加相关的工程师、产品经理等为参与者。他们马上就能直接看到图片，再评论中提出自己的修改意见。

版本控制

设计师吸纳了大家的意见之后，完成第一个修改稿。他只需要点击「更新」，不仅会提醒所有参与者，之前的版本也不会被删除——随时都可以找回预览、下载。

一个新功能从设想到发布，需要涉及到产品经理、设计部门、开发部门和市场部门。

设计师更加关心的是，不论是产品或者非产品的设计需求，都必须花费不少时间请需求方提供文档、背景资料等等。这些资料可能在不同项目的不同地方；找到他们绝非易事。
