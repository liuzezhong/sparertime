Teambition API FAQ

API 常见问题

能简单描述下Teambition API 的功能范围吗？

可以完成 Teambition 绝大部分的功能，通过人工值守+判断能完成的工作，基本都可以通过调用 Teambition 的 API 来自动化。

我们十分推荐使用 Teambition 来做工单 / 待处理事项 界面，或者用 Teambition 分项目的任务版、分享墙、日程表、文件库的内容，加上评论、关联等特性，来做相似用途的数据库。

其中比较常用和值得关注的部分为，Oauth 2.0 和 Webhooks

有可以参考的 Demo 吗？

我们在Github上上传了一个Demo,可以用一个自己实现的 简（丑）单（陋）界面来调整对应面板的任务条目。 供参考：https://github.com/teambition/openapi-examples/tree/master/todomvc

我是新手，Oauth 相关的内容好像比较复杂，有相应的参考资料可以入门吗？

我们推荐阮一峰的文章。 Teambition 对应的是其中最完整的 授权码 模式

有相关的 SDK 吗？

感谢 红宇 提供的Python 版的 SDK：https://github.com/jxtech/teambition-api

感谢 Aidi Stan 提供的 Ruby 版 SDK https://github.com/aidistan/ruby-teambition

API 请求有相关的频率限制吗？

连续5次以内的请求，没有严格间隔，超过5次，需要间隔500ms，以此类推。“连续”的认定标准是最多的请求间间隔为20s。

  • 500ms > 5
  • 1000ms > 9
  • 2000ms > 19
  • 4000ms > 29
  • 8000ms > 39
  • 16000ms > 49

我可以在不申请应用的情况下测试一下 API 吗？/ 为什么登录状态下访问 API 地址，没带 AccessToken也能正常访问？

登录用户系统会自动分配 token，能在图形界面完成的动作，都可以用我们的 Restful API 来完成。但是如果通过 Postman 等请求发送工具调试的时候，必须所有请求带上 access_token, token的获得请查看文档的 Oauth 2.0 部分。 token 放在headers或者query string里面,假设你的 access token 是 XXXXX，形式如下： headers={'Authorization': 'OAuth2 XXXXX'} 或者querystring里面带上 access_token=XXXXX

验证域和接口域分别是什么，怎么分工的？

认证域: https://account.teambition.com Teambition使用OAuth2 标准进行用户验证，获取 token 后，通过接口域进行操作。

接口域: https://api.teambition.com

Token 会过期么？

目前不会，但可以自行刷新。

如何查看 Webhook 事件列表？

浏览器登录情况下访问，http://api.teambition.com/api/projects/webhooks 可以查看目前支持的事件列表。

Webhook 的返回有事例么？

由于这块目前还在不断完善中，暂时没有文档化，返回值的示例可以通过登录后访问http://api.teambition.com/api/projects/webhooks/{webhook_id，如event.create} 来取得，Header 等信息请参考https://docs.teambition.com/wiki/webhooks 的内容，实际创建 Webhook 测试。

创建组织 Webhook 指定事件总是失败？

很抱歉，目前还未支持组织下事件的 Webhook 创建。

任务分组模板相关的 API 文档中好像没有找到？

任务分组的模板的新建、修改、和使用方面的 api 我们在调研和前期开发中。

一些请求，如调用/api/projects/{_projectId}/posts这个接口，如果带着后面:attachments=>[], :involveMembers=>["1234567890"], :tagIds=>[]这几个参数就会报错，是这几个参数有什么特殊限制吗？

请求的content-type需要注意一下，你把content-type设置为application/json试试， 如果不是，你那些array的参数有可能被认为是string。

API 的申请和使用需要付费版本吗？

不需要，但有些与组织相关的接口需要你拥有组织相关权限才能使用，组织功能可能会收费。当然，我们鼓励正常使用。

有集中的错误代码解释文档吗？

我们还在整理中。

有关评论部分的 API在哪里？

请参见文档的 “动态” 部分https://docs.teambition.com/wiki/activities 。

上传文件时，如何调用 striker 服务？

在https://docs.teambition.com/wiki/works#works-create 中可以看到提及相关服务， 目前文件上传相关的接口尚未开放。

接口实现类

团队成员的任务这个接口错误地返回了 Null ？https://docs.teambition.com/wiki/teams#teams-get_tasks_by_member

是这样的，https://docs.teambition.com/wiki/teams#teams-get_tasks_by_member 这里对应的方法，调取的只能是用户作为团队一员身份加入的项目当中，被分配到的任务，对应图形界面中https://www.teambition.com/organization/{organization_id}/teams/{team_id} 所看到的任务。 如果那个任务是员工以个人身份加入组织中的项目，领取到的，那么只能从个人任务相关的API 中获取.。
